【学习笔记】函数的反演变换

莫比乌斯反演

请先学习狄利克雷卷积，否则您会一脸懵逼。

一般式： \[ \begin{aligned} F(n) = \sum_{d | n} G(d) \rightarrow G(n) = \sum_{d | n} \mu(\frac{n}{d}) F(d) \\ F(n) = \sum_{n | d} G(d) \rightarrow G(n) = \sum_{n | d} \mu(\frac{d}{n}) F(d) \end{aligned} \]

证明可以考虑下面恒等式的第一条。

然后再放一些有助于反演的恒等式：（其中 \(*\) 指狄利克雷卷积，\(\epsilon(n)\) 表示 \([n = 1]\)） \[ \begin{aligned} &\mu * 1 = \epsilon \\ & \varphi * 1 = id \\ & \mu * id = \mu * 1 * \varphi = \epsilon * \varphi = \varphi \\ &\varphi(n) = \sum_{i = 1}^n \epsilon(\gcd(n, i)) \\ &\varphi(nm) = \frac{\varphi(n)\varphi(m)\gcd(n, m)}{\varphi(\gcd(n, m))} \\ &\sum_{i=1}^n\sum_{j = 1}^n \epsilon(\gcd(i, j)) = 2 \cdot (\sum_{i = 1}^n\varphi(i)) - 1 \\ & \sum_{d | ij} 1 = \sum_{x|i} \sum_{y|j} \epsilon(\gcd(x, y)) \\ & \sum_{d | ijk} 1 = \sum_{x|i} \sum_{y|j} \sum_{z|k} \epsilon(\gcd(x, y))\epsilon(\gcd(y, z))\epsilon(\gcd(x, z)) \\ &\cdots\cdots \end{aligned} \]

推完式子之后，可以考虑用一些数论筛法预处理某些项，或是用整除分块求值。

拿一道例题来讲解吧。

一共 \(T\) 组询问 \((T \le 10^4)\)，给定常数 \(K\)，每次询问单独给定 \(N\)，满足 \(N \le 10^7\)，求出： \[ \sum_{i = 1}^N\sum_{j = 1}^N(i + j)^K \gcd(i, j)\mu^2(\gcd(i, j))~(\bmod 2^{32}) \] 看模数 \(2^{32}\)，显然可以自然溢出，这可以帮助我们减小常数。

接下来表演推式子： \[ \begin{aligned} &\sum_{i = 1}^N\sum_{j = 1}^N(i + j)^K \gcd(i, j)\mu^2(\gcd(i, j)) \\ =& \sum_{d = 1}^N\sum_{i = 1}^N\sum_{j = 1}^N[\gcd(i, j) = d]((i + j)^K d\mu^2(d)) \\ =& \sum_{d = 1}^Nd^{K + 1}\mu^2(d)\sum_{i = 1}^{\lfloor\frac{N}{d}\rfloor} \sum_{j = 1}^{\lfloor\frac{N}{d}\rfloor} \epsilon(\gcd(i, j))(i + j)^K \\ =& \sum_{d = 1}^N d^{K + 1}\mu^2(d)\sum_{i = 1}^{\lfloor\frac{N}{d}\rfloor}\sum_{j = 1}^{\lfloor\frac{N}{d}\rfloor}(i + j)^K\sum_{x | \gcd(i, j)} \mu(x) \\ =& \sum_{d = 1}^Nd^{K + 1}\mu^2(d)\sum_{x = 1}^{\lfloor\frac{N}{d}\rfloor}x^K\mu(x)\sum_{i = 1}^{\lfloor\frac{N}{dx}\rfloor}\sum_{j = 1}^{\lfloor\frac{N}{dx}\rfloor}(i + j)^K \\ =& \sum_{T = 1}^NT^K\sum_{i = 1}^{\lfloor\frac{N}{T}\rfloor} \sum_{j = 1}^{\lfloor\frac{N}{T}\rfloor}(i + j)^K \sum_{d|T}d\mu^2(d)\mu(\frac{T}{d}) \end{aligned} \] 到这里式子就是可做的了。

我猜你现在肯定一脸懵。

那就先来讲一讲吧。

  • \(Step~1\)

我们考虑枚举这个 \(\gcd\) 的值。

为什么这里要做这样一个看似只会添一层循环的无意义举动呢？

可以发现，在后面的式子中多出了一个艾弗森括号，这个括号里写下的等式是可以转化为单位元的形式的。

  • \(Step~2\)

我们考虑对于每一个 \(d\)，只枚举 \(d\) 的倍数。

（实际上，此时枚举的 \(i, j\) 在后面的运算中会乘上一个 \(d\)。）

这样，艾弗森括号中的等式就可以被改写了，具体来说就是 \([\gcd(id, jd) = d] =[\gcd(i, j) = 1]\)。

在上面的式子中，后者已经写成了单位元的形式。

值得注意的是，由于 \(i, j\) 实际上除去了一个 \(d\)，所以运算中多出来的一个 \(d^K\) 要提到前面去。同时只和 \(d\) 有关的一项 \(d\mu^2(d)\) 也要提到前面去。

  • \(Step~3\)

还记得上面的公式 \(\mu * 1 = \epsilon\) 吗？我们可以直接把这个单位元转化为莫比乌斯函数和 \(1\) 的狄利克雷卷积！

所以，把卷积用好了，莫比乌斯反演就不难了，再难也只可能难在函数的筛法上。

然后呢？我们再把这个枚举的 \(x\) 提前，把 \(i, j\) 再次强制转化为 \(dx\) 的倍数，多出来的 \(x^K\) 同 \(\mu(x)\) 一起往前提。

继续，我们发现这个式子代数还是太多了，于是考虑枚举 \(T = dx\)，那么 \(d\) 必然是 \(T\) 的因数，此时的 \(x\) 则恰好等于 \(\frac{T}{d}\)，这样化简出来，式子已经可做了。

问题是，我们怎样知道式子是可做的呢？

我们设 \(S(N) = \sum_{i = 1}^N\sum_{j = 1}^N(i + j)^K\)，\(f(T) = \sum_{d|T}d\mu^2(d)\mu(\frac{T}{d})\)。

现在整个式子化简成了这样一个形式：\(\sum_{T = 1}^N T^KS(\lfloor\frac{N}{T}\rfloor)f(T)\)。

接下来分别考虑线性预处理 \(S\) 和 \(f\)。

可以发现在 \(S(N)\) 中，从 \(2 \sim 2N\) 的每一个数 \(d^K\) 一共被计算了 \(\min(d - 1, 2N + 1 - d)\) 次。

于是我们可以再设两个辅助函数 \(F(N) = \sum_{i = 1}^N i^K\)，\(G(N) = \sum_{i = 1}^N F(i)\)，显然这两个函数都是可以线性预处理的。

那么我们有式子：\(S(N)=\sum_{i=N+1}^{2N}F(i)-\sum_{i = 1}^N F(i) = G(2N)-2G(N)\)，同样也可以线性预处理。

再来看 \(f\)，可以发现 \(f\) 是一个狄利克雷卷积的标准形式，并且卷的是两个积性函数，说明整个式子也是一个积性函数。如果您阅读过我的文章《积性函数的筛法》，可以知道这样的函数是可以欧拉筛的。

我们来讨论一下 \(f\) 在 \(p^k~(p \in \operatorname{Prime})\) 处的取值吧。

  • \(k = 0\)，显然有 \(f(1) = 1\)；
  • \(k = 1\)，有 \(f(p) = \mu(p) + p\mu^2(p) = p - 1\)；
  • \(k = 2\)，有 \(f(p^2) = \mu(p^2) + p\mu^2(p)\mu(p) + p^2\mu(p^2) = -p\)；
  • \(k \ge 3\)，则每一项都存在 \(\mu\) 为 \(0\) 的情况，答案为 \(0\)。

次数很好判断，所以分讨即可。

最后，可以发现 \(\frac{N}{T}\) 的取值个数是 \(\sqrt{N}\) 级别的，所以可以套用整除分块。同时为了方便，可以把 \(T^K\) 这一项扔进 \(f\) 中一起预处理（最后求前缀和时配合 \(F\) 数组一起求即可）。

为了省空间，把 \(F\) 和 \(G\) 当作一个数组，毕竟有了 \(G\)，\(F\) 也没什么用了。

二项式反演

其实二项式反演有一个更不广为人知的名字——广义容斥原理。

一般有两种形式： \[ \begin{aligned} f(n) = \sum_{i = 0}^nC_n^ig(i) \Leftrightarrow g(n) = \sum_{i = 0}^n(-1)^{n - i}C_n^if(i) \\ f(n) = \sum_{i = n}^mC_i^ng(i) \Leftrightarrow g(n) = \sum_{i = n}^m(-1)^{i - n} C_i^nf(i) \end{aligned} \] 正如前面所说，我们一般设 \(g(n)\) 为 “恰好满足 \(n\) 个性质” 的方案数。对于形式一和形式二，我们一般设 \(f(n)\) 为 “至多满足 \(n\) 个性质” 和 “至少满足 \(n\) 个性质” 的方案数。

显然 \(f(n)\) 更好求，并且 \(g(n)\) 一般都做为题目要求的东西。

例题可以康康这道，太板了就不放题解和代码了。

单位根反演

其实就是这样一个式子：\([n | k] = \frac{1}{n}\sum_{i = 0}^{n - 1} \omega_n^{ik}\)。

\(\omega\) 指单位根。从数学上讲，它是一个复数。但是从广义上讲，只要在给定的数域和运算内满足 \(\omega^n_n\)（即 \(n\) 阶单位根的 \(n\) 次方）为 \(1\)，那么它就可以称做单位根。所以 \(\omega\) 实际上还可以是原根。

证明略去 ，显然是因为不会。

康康例题

首先我们有 \(\lfloor\frac{i}{k}\rfloor = (\sum_{j = 0}^i[k | j]) - 1\)，这样就转化为单位根反演的形式了。

（可以不管后面的减 \(1\)，因为可以直接拆开然后二项式定理搞掉。）

然后我们直接莽式子： \[ \begin{aligned} &\sum_{i = 0}^n C_n^i p^i ((\sum_{j = 0}^i [k | j]) - 1) \\ =&(\sum_{i = 0}^nC_n^i p^i\sum_{j = 0}^i\frac{1}{k}\sum_{t = 0}^{k - 1}\omega_k^{jt}) - (p + 1)^n \\ =&\frac{1}{k}(\sum_{t = 0}^{k - 1}\sum_{i = 0}^n C_n^i p^i\sum_{j = 0}^i(\omega_k^t)^j) - (p + 1)^n \\ =&\frac{1}{k}(\sum_{t = 0}^{k - 1}\sum_{i = 0}^n C_n^i p^i\frac{\omega_k^{t(i + 1)} - 1}{\omega_k^t - 1}) - (p + 1)^n \\ =&\frac{1}{k}(\sum_{t = 0}^{k - 1}\frac{(\sum_{i = 0}^n C_n^i p^i \omega_k^{t(i + 1)}) - (p + 1)^n}{\omega_k^t - 1}) - (p + 1)^n \\ =&\frac{1}{k}(\sum_{t = 0}^{k - 1}\frac{(\omega_k^t(p\omega_k^t + 1)^n- (p + 1)^n}{\omega_k^t - 1}) - (p + 1)^n \end{aligned} \] 可以发现，上面的式子中大量地用到了二项式定理，这玩意确实会经常和单位根反演配合使用。

上面的式子已经可做了，时间复杂度为 \(O(k \log_2 n)\)，带 \(\log\) 是因为快速幂。

然后您会发现您连样例都过不去。

我们再来审视一下整个式子。可以发现，当 \(t = 0\) 时，\(\omega_k^t = 1\)，此时分母为 \(0\)，这一段贡献是算错了的。

我们回到原来的式子：\(\sum_{i = 0}^n C_n^i p^i\sum_{j = 0}^i(\omega_k^t)^j\)，当 \(\omega_k^t = 1\) 时，该式子为 \(\sum_{i = 0}^n C_n^i p^i(i + 1)\)。

我们考虑一下这个式子怎么做。

首先我们把后面的加 \(1\) 提出来，直接做一遍二项式定理。

剩下的就是 \(\sum_{i = 0}^n C_n^i p^ii\)。

然后我们有一个广为人知的组合恒等式：\(iC_n^i = nC_{n - 1}^{i - 1}\)。

证明：\(i C_n^i = i \cdot \frac{n!}{i!(n - i)!} = \frac{n!}{(i - 1)!(n - i)!} = n \cdot \frac{(n - 1)!}{(i - 1)!((n - 1) - (i - 1))!} = n C_{n - 1}^{i - 1}\)。

由于 \(i = 0\) 时没有贡献，因此我们从 \(i = 1\) 开始枚举。 \[ \begin{aligned} &\sum_{i = 1}^n C_n^i p^ii \\ = &n\sum_{i = 1}^nC_{n - 1}^{i - 1} p^i \\ = &n \sum_{i = 0}^{n - 1} C_{n - 1}^i p^{i +1} \\ = &np\sum_{i = 0}^{n - 1}C_{n - 1}^i p^i \\ = & np(p + 1)^{n - 1} \end{aligned} \] 算法到此就大功告成了。

快速沃尔什变换

\(\operatorname{FWT}\)，又称废物树，主要用于处理子集之间的关系。

用例题来讲：

给定两个长度为 \(2^n\) 的数组 \(A, B\)，设： \[ C_{i} = \sum_{j \otimes k = i} A_j\cdot B_k \] 求当 \(\otimes\) 为 \(\operatorname{or}, \operatorname{and}, \operatorname{xor}\) 时的 \(C\)。\((n \le 17)\)

解法：

对于 \(\operatorname{or}\)，我们考虑构造 \({F}'_i = \sum_{j \operatorname{or} i = i} F_j\)。

为什么要这样构造？

因为有： \[ \begin{aligned} A' \times B' &= \sum_{j \operatorname{or} i = i} A_j \sum_{k \operatorname{or} i = i} B_k \\ &= \sum_{j \operatorname{or} i = i} \sum_{k \operatorname{or} i = i} A_j B_k \\ &=\sum_{(j \operatorname{or} k) \operatorname{or} i = i} A_jB_k \\ &= C' \end{aligned} \] 现在我们考虑如何将 \(A \to A'\)。

其实就是一个类似 \(\operatorname{CDQ}\) 分治的思想。

可以发现，在原数组的基础上，每一个数都会对自己二进制表示下的真超集造成贡献，而真超集的特点在于：至少存在一位，满足本身为 \(0\) 而超集为 \(1\)。

所以做法就很显然了：我们类似快速傅里叶变换的递归过程，从小到大枚举位数，把该位上为 \(0\) 的位置贡献到该位为 \(1\) 位置。

而逆变换怎么办呢？直接减回来呗。

可以发现两个变换可以合并：

与的构造和变换同理：

异或的变换就有点麻烦了，因为存在交叉贡献。

我们定义 \(a \otimes b = \operatorname{popcount}(a\operatorname{and} b)~\bmod~2\)，其中 \(\operatorname{popcount}(k)\) 表示 \(k\) 在二进制下 \(1\) 的个数。

那么我们存在一个性质：\((i \otimes j) \operatorname{xor} (i \otimes k) = i \otimes (j \operatorname{xor} k)\)。

证明的话，考虑化原式为 \(\operatorname{popcount}(i \operatorname{and} j) + \operatorname{popcount}(i \operatorname{and} k) \equiv \operatorname{popcount}(i \operatorname{and} (j \operatorname{xor} k))~(\bmod~2)\)，然后按位分讨即可，这里就不详细写了。

然后我们构造 \(F'_i = \sum_{i \otimes j = 0}F_j- \sum_{i \otimes j = 1}F_j\)。

那么有： \[ \begin{aligned} A' \times B' &= (\sum_{i \otimes j = 0}A_j- \sum_{i \otimes j = 1}A_j)(\sum_{i \otimes k = 0}B_k- \sum_{i \otimes k = 1}B_k) \\ &= (\sum_{i \otimes j = 0}A_j)(\sum_{i \otimes k = 0}B_k) - (\sum_{i \otimes j = 0}A_j)(\sum_{i \otimes k = 1}B_k) - (\sum_{i \otimes j = 1}A_j)(\sum_{i \otimes k = 0}B_k) + (\sum_{i \otimes j = 1}A_j)(\sum_{i \otimes k = 1}B_k) \\ &=\sum_{i \otimes (j \operatorname{xor} k) = 0} A_jB_k - \sum_{i\otimes (j \operatorname{xor} k) = 1} A_jB_k \\ &= C' \end{aligned} \] 现在的问题就在于计算贡献。

还是考虑从小到大枚举每一位 \(i\)。

我们设 \(F'_j\) 为枚举到的位置 \(j\) 的 \(F'\) 的值，\(F'_{j_0}\) 表示的是 \(j\) 的前 \(i - 1\) 位和第 \(i\) 位的 \(0\) 组成的数的 \(F'\) 值，\(F'_{j_1}\) 同理。（注意前面的 \(F'_j\) 是要求的，后面的 \(F'_{j_0},F'_{j_1}\) 是已经求出来的）

如果 \(j\) 第 \(i\) 位本身为 \(0\)，那么在 \(\operatorname{and}\) 意义下这一位必然为 \(0\)，所以这个位置不会对 \(\otimes\) 的结果造成影响。也就是说，其贡献就等于 \(j_0\) 和 \(j_1\) 这两个位置的数值和，即 \(F'_j = F'_{j_0} + F'_{j_1}\)。

如果 \(j\) 第 \(i\) 位本身为 \(1\)，由于 \(j_0\) 的第 \(i\) 位为 \(0\) ，所以它不会改变 \(\otimes\) 的结果，即 \(F'_{j_0}\) 会对 \(F'_{j}\) 产生正贡献，而 \(j_1\) 的第 \(i\) 位为 \(1\)，导致 \(\otimes\) 结果改变，所以 \(F'_{j_1}\) 会对 \(F'_{j}\) 产生负贡献，即 \(F'_j = F'_{j_0} - F'_{j_1}\)。

如果你学过小学奥数就可以发现逆变换就是在原来的基础上乘上了一个 \(\frac{1}{2}\)。

有了 \(\operatorname{FWT}\)，我们还可以做一件事情——子集卷积：

给定序列 \(A\) 和 \(B\)，序列 \(C\) 满足： \[ C_k = \sum_{i \operatorname{and} j = 0, i \operatorname{or} j = k} A_i B_j \] 求出 \(C\)。

题解：

对于限制 \(i \operatorname{or} j = k\)，废物树可以直接干。

对于限制 \(i \operatorname{and} j = 0\)，我们发现可以转化为 \(|i| + |j| = |i \operatorname{and} j|\)。

所以，我们对每个集合大小跑一遍 \(\operatorname{FWT}\)，然后计算的时候同时用两层循环枚举集合大小即可，最后再跑 \(\operatorname{IFWT}\)。

子集反演

实际上，子集反演的前置知识就是上面提到的废物树和子集卷积。

它其实就是类似这样的一个式子： \[ F(S) = \sum_{T \subset S} G(T) \Leftrightarrow G(S) = \sum_{T \subset S} (-1)^{|S| - |T|} F(T) \]

例题

我们设 \(F(S)\) 为有多少种方案使得玩具的集合是 \(S\) 的子集，设 \(f(S)\) 为有多少个盒子的玩具集合是 \(S\) 的子集，那么显然有 \(F(S) = 2^{f(S)} - 1\)（减不减 \(1\) 无所谓，反正最后会容斥掉）。再设 \(G(S)\) 为有多少种方案使得玩具集合恰好为 \(S\)，那么可以直接套用上面的子集反演公式。

（事实上，大多数计数题也用到了这个设置状态的思路，即设置 “至少”、“至多” 和 “恰好”。）

然后考虑求 \(f(S)\)，可以发现这就是一个废物树的变换，直接套上即可。

（这里提一句，\(f(S)\) 实际上称作高维前缀和，而这种用 \(\operatorname{FWT}\) 优化其的方法称做 \(sos\underline{}dp\)，又叫求救优化。）
